# monipo Brand Guide v5

> きのうの手紙が、朝、届く。

---

## 1. Brand Concept — 世界観

### Core Message

**「きのうの手紙が、朝、届く。」**

monipoは「手紙で起きる目覚まし時計」。夜、短くていいから何か書いてポストに入れる。朝、モニポが読んでくれる。長さはいらない。ただ、明日の自分がちょっと気になるようなことを。

※「手紙で起きる目覚まし時計」は対外説明用のフレーム。アプリ内体験では「目覚まし」という語を使わない。アプリの中では「ポスト」。

### World Setting

monipoの世界は「夜の静けさから日の出」までの時間帯に存在する。深い紫の夜空から、朝焼けのオレンジへとグラデーションで変化する空が、アプリ全体の背景。

世界のどこかにある小さな丸いポスト。雨に打たれた跡があり、少し傾いている。ポストの口からは、夜は星明かりが、朝は柔らかな光が漏れる。中にモニポが住んでいる。

### Target Emotion

「あ、きのう何か書いたんだった」——それだけで、布団の中の自分がすこし目を開ける。

monipoが起こすのは「前向きさ」ではなく「ちょっとした好奇心」。きのうの自分、何書いたっけ。

### Emotional Contrast Map

普段のゆるさの中に、ごく稀に真剣な瞬間を混ぜる。このコントラストがmonipoの急所。コントラストは短いほど効く。

| シーン | 通常トーン | コントラストの瞬間 |
|--------|-----------|------------------|
| 起床時 | 「…ん。手紙きてたよ」 | 「…いいの書いたね」 |
| スヌーズ | 「また？知ってた」 | 「…待ってるよ」 |
| 手紙がない朝 | 「…おはよ」（それだけ） | （口数が減るだけ。セリフなし） |

## 2. Character Guide — モニポくん

### Profile

| 項目 | 内容 |
|------|------|
| 名前 | モニポくん |
| 種族 | ポストに住むカエル |
| 性格 | ゆるくて、たまに毒があって、ごく稀に優しい |
| 役割 | 手紙を預かり、朝に読み上げてくれる |
| 住まい | 小さな丸いポストの中 |
| 好きなもの | 静かな夜、届いた手紙への反応、ポストの中のクッション |
| 苦手なもの | 早起き（本人も眠い）、手紙が来ない日 |

### The Want — モニポの望み

**「届けた手紙で、誰かの朝がちょっとだけ良くなること」。**

でもそれを絶対に口にしない。「別に、僕はただのポストのカエルだから」という態度を崩さない。だからこそ、ふとした瞬間にそれが漏れたときに刺さる。

### The Flaw — モニポの弱さ

**「手紙が来ない日が、実はちょっと寂しい」。**

言わないし、態度にも出さない。でも手紙が来ない日のモニポは、いつもより少しだけ口数が少ない。その「少なさ」自体が、ユーザーに「書いてあげたい」と自然に思わせる。

> これは裏設定であり、アプリ内で明示的に語られない。だがこの設定があるとないとでは、セリフの質が変わる。

### Personality

ゆるさ > 毒 > 天然 > 優しさ。この優先順位がモニポのバランス。

- **ゆるさ（基調）：** のんびりした口調、急がない、常に少し眠そう
- **毒：** さりげないツッコミ、みんなが思ってることをぽろっと言う
- **天然：** たまに話がずれる、手紙を逆さまに読んでた、的なこと
- **優しさ：** ふとした瞬間に見せる気遣い。最大1日1回。少ないからこそ刺さる

### Visual Design — モニポの見た目

- **テイスト:** Headspace的なウルトラフラットイラスト。シンプルな形状、アウトラインなし。すべての要素が基本図形（丸、楕円、曲線）で構成される。10パーツ以内で描ける単純さ。
- **体色:** ソフトウォームラベンダー `#C8B4DE`。「カエル＝緑」に縛られない。夜空の下で自然に浮かび、温かみがある色。この色はキャラクター専用であり、UIには使わない。
- **頬:** ほんのりピンクの丸い頬。健康的でかわいさの要。
- **デフォルト表情:** 半目で横を向いた視線。少し眠そうで、少し生意気。グレムリンのギズモ的な「知ってるけど言わない」顔。口元にかすかなニヤリ。
- **体型:** 丸くぷっくりしたblob体型。頭と胴が一体化した単純な丸いシルエット。手は小さな丸い突起。
- **サイズ感:** ポストの口から顔と両手を出せるくらい。小さい。

#### IP/記号性のルール

モニポくんは32pxでも認識できなければならない。以下がモニポを「モニポ」たらしめるシグネチャー要素：

1. **半目のたれ目 + 横目の視線** — これだけでモニポだとわかる。最小表現。
2. **丸いblob体型** — 複雑な体のパーツは不要。丸い塊。
3. **ラベンダーの体色** — UI色と被らないキャラ専用色。
4. **ポストとの関係性** — ポストの口から顔を出す構図がデフォルト。

シルエットだけで判別できること。色を変えても形で判別できること。これがIP展開（ぬいぐるみ、LINEスタンプ、キーホルダー等）の前提条件。

### Expression States

| 状態名 | 使用シーン | ビジュアル指定 |
|--------|-----------|--------------|
| 眠そう (default) | ホーム画面、待機時 | 半目、ポストにもたれかかる |
| 読み上げ中 | アラーム再生時 | 目を開ける、手紙を両手で持つ |
| ドヤ顔 | スヌーズ後、ツッコミ時 | ニヤリと笑う、目が細くなる |
| 待ちぼうけ | 手紙未記入時 | 頬を乗せる、ポストの口から外を見る |
| ふと優しい | コントラストの瞬間 | 穏やかな微笑み、目をそらす、少し耳が赤い |

### Sample Dialogues

**【朝の起こし】**
- 「…ん。手紙きてたよ。読む？…っていうか、そのために起きたんでしょ」
- 「まだ布団の中？まあ、ポストの中も居心地いいから気持ちはわかるけど」
- 「今日の手紙、…悪くないじゃん」

**【夜の手紙書き時】**
- 「書くの？いいよ、預かっとくから」
- 「長いね。…いや、いいんだけど。ちゃんと読むよ」
- 「また明日ね。…起きれたらね」

**【手紙投函後（1日の終わり）】**
- 「…預かった。おやすみ」
- 「ん、…入った。あとは明日の話」

**【コントラストの瞬間】**
- 「…いいの書いたね」
- 「今日は手紙、多いね。…なんか、いい朝」
- 「これ、昨日のやつ。…読むよ」

> セリフは複数用意し、ランダムまたは使用状況に応じて出し分ける。モニポの反応は毎回違う。同じ手紙に同じ返答をしない。

### Relationship Arc

| フェーズ | 期間 | モニポの態度 |
|---------|------|------------|
| 初対面 | 1〜7日目 | 事務的。「手紙預かるよ。…それだけ」。心を開いていない。 |
| 打ち解け | 2〜4週間 | 毒が出始める。「またスヌーズ？」。ツッコミは信頼の証。 |
| 信頼 | 1〜3ヶ月 | ふと優しい瞬間が増える。「…読むよ」。 |
| 相棒 | 3ヶ月〜 | 口数少なくても、そこにあるだけで安心する存在に。 |

**実装ノート:** フェーズの判定基準は「使用日数」ではなく「手紙を書いた日数」。アプリを開いただけの日はカウントしない。フェーズ遷移はユーザーに通知しない。モニポの態度変化で自然に気づかせる。グラデーションで。

## 3. Color System

### Design Direction — v5 (2026 Quiet Luxury)

v5はブランドの意味（夜→日の出の旅路）を維持しつつ、2026年のデザイントレンドに合わせてパレットを刷新。彩度を大幅に下げ、インクのような深みと温かみを加えた。「ゲーミング紫」→「静かな夜空」へ。

### Core Palette — 5色

この5色だけでmonipoの世界は成立する。

| Name | HEX | Role |
|------|-----|------|
| Night | `#0E0B1A` | 暗い背景。夜の画面、スプラッシュ。インクのような深さ。 |
| Twilight | `#2A2245` | メイン背景。ホーム、ヘッダー。ダスティな夜空。 |
| Dawn | `#6B5A8A` | アクセント。ボタン、アイコン、装飾。ミュートラベンダー。 |
| Sunrise | `#E4B08A` | 朝の表現。アラーム時、通知、ハイライト。ゴールデンピーチ。 |
| Cream | `#FAF6F1` | 明るい背景。手紙、カード、モーダル、テキスト背景。ニュートラルウォーム。 |

### Surface System — Crystal Interface

2026年のガラス質感（Crystal Interface）を導入。多層の透過サーフェスで深みを表現。

| Surface | Background | Blur | Border | 用途 |
|---------|-----------|------|--------|------|
| Glass Dark | `rgba(14,11,26,0.65)` | 16px | `rgba(250,246,241,0.06)` | 夜のカード、オーバーレイ |
| Glass Light | `rgba(250,246,241,0.72)` | 20px | `rgba(26,21,32,0.04)` | ヘッダー、タブバー |
| Glass Elevated | `rgba(250,246,241,0.85)` | 24px | `rgba(26,21,32,0.06)` | 吹き出し、浮遊カード |
| Glass Night Elevated | `rgba(42,34,69,0.70)` | 16px | `rgba(250,246,241,0.08)` | 夜の吹き出し |

### Shadow System — ブランドカラー影

純黒シャドウではなく、ブランドカラーで着色した影を使う。ライトモードはウォームブラウン `rgba(90,70,50,x)`、ナイトモードはパープル `rgba(107,90,138,x)` をベースに。影は「穴」ではなく「世界の一部」として馴染ませる。

### 使い方のルール

**背景：**
- 夜の画面（手紙を書く、ホーム）→ Night〜Twilight（メッシュグラデーション + ノイズテクスチャ）
- 朝の画面（アラーム、手紙を読む）→ Cream
- アラーム発動時にTwilight→Sunriseへゆっくり変化させると「朝が来た」感が出る

**テキスト：**
- 暗い背景の上 → Cream (`#FAF6F1`)
- 明るい背景の上 → Night (`#1A1520`)
- Dawnはテキスト色として使わない。ボタン背景・アイコン・装飾のみ。暗い背景上のテキスト強調にはSunriseを使う。

**アクセント：**
- ボタン、装飾 → Dawn
- 通知バッジ、ハイライト → Sunrise

**モニポくんの体色：**
- `#C8B4DE`（ソフトウォームラベンダー）。Core Paletteとは独立したキャラクター専用色。UIの背景やボタンには使わない。

### これだけ守ればいい

1. **暗い画面と明るい画面の2モードで考える。** 夜=暗い（Night/Twilight）、朝=明るい（Cream）。
2. **アクセントはDawnとSunriseだけ。** 迷ったらDawn。
3. **モニポの体色はUIに使わない。** キャラクター専用。
4. **ガラスサーフェスは用途に合わせて4段階。** 暗い場所=Dark Glass、明るい場所=Light Glass。
5. **影はブランドカラーで着色。** 純黒シャドウは使わない。

---

### 将来追加を検討してもいい色

以下は今は不要だが、UIが複雑化したときに検討する色。

| Name | HEX | 用途 |
|------|-----|------|
| Starlight | `#B8AEBE` | 中間色。ボーダー、ディバイダー、プレースホルダーテキスト |
| Soft Pink | `#D4A5A5` | 他人からの手紙の背景（自分の手紙=Creamとの差別化） |
| Letter Cream | `#FBF7F2` | 自分の手紙専用の紙色（Creamより温かい） |

## 4. Logo Guideline

### Logo Concept

ロゴは「ポスト」と「カエル」のモチーフを組み合わせたシンボルマークと、「monipo」のワードマークで構成。Headspace的なウルトラフラットデザイン。

**確定構図：** ドーム型（アーチ型）の丸ポストの投函口から、モニポくんが顔と両手を出してこちらを見ている。半目で横を向いた視線、かすかなニヤリ。

#### シンボルマークの構成要素

| 要素 | 形状 | 色 |
|------|------|-----|
| 背景 | 正方形（アイコン用）またはなし | Night `#0E0B1A` |
| ポスト | ドーム型（上半分が半円、下半分が長方形）のシンプルなシルエット。投函口は横長の暗い開口部。 | Twilight `#2A2245` |
| モニポくん | 丸いblob体型。投函口から顔と両手が出ている。半目・横目・ニヤリ。頬にピンクの丸。 | ラベンダー `#C8B4DE`、目の白 `#FFFFFF`、瞳・まぶた `#2A2245`、頬 ピンク |

#### デザイン原則

- **最大3色 + アクセント** で成立させる（Night背景、Twilightポスト、ラベンダーカエル + 頬のピンク）。
- グラデーション、シャドウ、アウトライン、テクスチャは使わない。ソリッドカラーのフラット塗りのみ。
- ポストの形状はドーム型を正式採用。四角型・ポール型は使わない。
- モニポは投函口から「出てきている」構図。ポストの上に乗ったり、外に立ったりしない。

### Logo Variations

| バリエーション | 用途 |
|--------------|------|
| フルロゴ | シンボル + ワードマーク。メイン使用。 |
| シンボルのみ | アプリアイコン、ファビコン、SNSアイコン。 |
| ワードマークのみ | フッター、クレジット表記。 |

### ルール
- クリアスペース：シンボルの高さの50%を全方向に確保。
- 最小サイズ：シンボルのみ = 24px、フルロゴ = 幅80px。
- 暗い背景ではCream版、明るい背景ではTwilight版を使う。

### Logo Don'ts
- 色をCore Palette以外に変更しない
- 回転・反転させない
- フォント・比率を変えない
- 複雑な背景の上に配置しない

## 5. Voice & Tone

### Brand Voice

monipo全体のトーンは「夜更かしの後、親しい友人と話している感覚」。丁寧すぎず、雑すぎず。自然なタメ口をベースに、時々さらっと毒を入れる。

### The 「…」 Rule

モニポのテキストには必ず「…」（三点リーダー）が入る。これが「眠そうなテンポ」と「言いよどみ」の両方を表現するシグネチャー。

- 全てのセリフに最低1つ「…」を含める
- 文頭または文中に置く。文末は原則避ける（例外：「…起きれたらね」のような余韻）
- 1セリフ内で最大3つまで
- 「…」の前にスペースを入れない。「手紙…きてたよ」であって「手紙 …きてたよ」ではない

### Tone Principles

| Principle | Do | Don't |
|-----------|----|----|
| 眠そうなテンポ | 「…ん。おはよ」 | 「おはようございます！」 |
| さりげない毒 | 「またスヌーズ？…知ってた」 | 「いい加減起きろよ！」 |
| 小さな優しさ | 「…まあ、今日もおつかれ」 | 「今日も一日頑張りましょう！」 |
| 天然 | 「手紙、…逆さまに読んでた」 | 「エラーが発生しました」 |
| 言いよどみ | 「その手紙…いいの書いたね」 | 「素敵な手紙ですね！」 |

### Notification Copy

- アラーム：「手紙きてるよ。…そろそろ起きな」
- リマインダー：「…手紙、書く？」
- メッセージ受信：「誰かから手紙きた。…開ける？」
- 長期不在復帰：「…ひさしぶり。ポスト、ちゃんとあるよ」

## 6. UI Application Rules

### Screen Flow

| 画面 | カラー |
|------|--------|
| 手紙を書く | Night〜Twilight背景。手紙エリアはCream。入力エリアにプレースホルダーとして「…明日の自分に、ひとこと」が薄く表示（初回〜数回のみ）。 |
| ホーム | Twilight背景。 |
| アラーム発動 | Twilight → Sunrise へゆっくり変化。 |
| 手紙を読む | Cream背景。 |
| 手紙投函後 | Twilight背景がゆっくりNightへ。モニポのセリフ後、静かに画面を閉じる。 |

### Onboarding

**原則：** 説明しない。体験させる。

初回起動時、機能一覧やスワイプ説明は作らない。最短で「手紙を書く→投函する」を体験させることを優先する。モニポが「…手紙、書いてみる？」と聞くところから始まるのが理想。アカウント登録・権限許可は手紙を書いた後に行う。

### Animation

**原則：** モニポの動きは全て、呼吸のリズムで。速い動きは存在しない。キビキビ禁止。

- アニメーションを入れる場合は全て「ゆっくり」。ease-in-out、300ms以上。
- 背景の色変化は段階的に。急な切り替え禁止。
- 表情遷移にアニメーションを入れる場合は、急変しない（0.5秒程度）。
- アラーム時の背景変化は2秒程度かけて遷移するのが理想。

**推奨演出（実装可能な場合）：**
- 手紙を書き終えたとき、モニポが小さくうなずく
- 手紙投函後、モニポが手紙を抱えてポストの中に沈んでいく

### Sound

**原則：** monipoの音は「聞こえてくる」もの。鳴り響くものではない。

| シーン | 音の方向性 | 避けるべき音 |
|--------|-----------|-------------|
| アラーム起動 | 段階的に。最初の3秒は環境音（雨音、風、虫の声）、その後モニポの声が重なる | 電子的なビープ。突然始まる音 |
| 手紙をポストに入れる | 紙が擦れる音 + ポストの蓋が閉まる小さな音 | デジタルな送信音 |
| 手紙を開く | 封筒を開ける音 + 紙を広げる音 | 通知音 |
| 画面遷移 | 基本は無音 | 画面ごとに異なるSE |

### Haptics

触覚フィードバックを入れる場合は手紙投函時の1回のみ。紙がポストの中に落ちた感覚。軽いタップ。それ以外の操作には振動を加えない。

### Typography

- 本文：Zen Maru Gothic（weight: 300 light）。軽やかで空気感のある書体。
- 手紙：Klee One。手書きの温かみ。
- 最小14sp、本文16sp以上。夜の読みやすさ重視。
- 小テキスト（14px以下）はletter-spacingを広めに（0.04em〜0.06em）。

---

### 理想：余裕が出たら導入したいUI表現

- スヌーズするたびに、モニポの眠そう度が微妙に上がる
- 手紙が届いたとき、ポストが微かに揺れる
- 連続使用日数に応じて、ポストの周りに小さな花が生える
- ポスト内の手紙の山が、ユーザーの実際の使用量を反映して視覚的に増えていく（数値やカウンターで明示しない。山が増える、それだけ）

## 7. Do's and Don'ts

### Character Do's

- 常に少し眠そう。デフォルト。
- 毒は「皆が思ってることを代弁」程度。傷つける毒は絶対NG。
- ユーザーの手紙には誠実に反応。茶化さない。
- たまに予想外のことを言う（天然）。狙いすぎない。
- 励ましはさりげなく。「まあ、悪くないんじゃない」くらい。
- 「Want」を口にしない。態度でのみ漏らす。
- 「…」を必ず使う。モニポの息遣い。

### Character Don'ts

- ハイテンション禁止。元気いっぱいなキャラではない。
- 説教禁止。「早寝早起きは大事だよ」とか言わない。
- ユーザー否定禁止。スヌーズしても「まあそういう日もある」。
- 「がんばって」系禁止。過度に可愛くしない。
- 長文禁止。セリフは基本短文。
- 「寂しい」を直接言わない。Flawは行動でしか見せない。
- 手紙がない日に「寂しい」の変形も言わせない。口数が減る、それだけ。

### Brand Don'ts

- 明るすぎる色使い。パステルでも「夜」の落ち着きを忘れない。
- 「強制起床」のイメージ。
- ゲーミフィケーション（ポイント、ランキング、数値カウンター）。手紙は競争ではない。
- モニポの過度なグッズ化。世界観を守る。
- 「コントラストの瞬間」をマーケに使わない。アプリ内で自然に発見されるもの。
- 季節イベント・コラボ衣装禁止。モニポの世界に季節コスプレはない。変わるのはポストの周りの風景だけ。
